DWH & OLAP Refresh

A structured technical roadmap for the modern data ecosystem.

Module 1

Architectural Foundations

Core methodologies for data organization and delivery.

Kimball vs. Inmon

  • Kimball: Dimensional Modeling focusing on query performance and usability.
  • Inmon: Normalized (3NF) central repository focusing on data integrity.

Modern Data Loading

The transition from ETL to ELT allows the use of cloud-native compute (Snowflake, BigQuery) to transform data after loading, maximizing scalability.

Module 2

OLAP Mechanics

Logical operations for multidimensional data analysis.

Mode Operation Best For
MOLAP Multidimensional (Cubes) Aggregated high-speed access.
ROLAP Relational (Live SQL) Granular, real-time drill-through.

The 5 Core Operations

  • Roll-up / Drill-down: Changing levels of granularity (e.g., Year vs. Month).
  • Slice / Dice: Selecting specific dimensions or sub-cubes.
  • Pivot: Rotating data for a different analytical perspective.
Module 3

Modern Trends (2026)

Current industry shifts toward decentralization and efficiency.

The Data Lakehouse

Unifying the Data Lake (cost-effective storage) and Data Warehouse (performance/ACID) using open table formats like Apache Iceberg.

Semantic Layers & Mesh

  • Headless BI: Centralized metrics logic (dbt/Cube) consumed by any BI tool.
  • Data Mesh: Decentralized ownership where domains own their data products.
Module 4

Advanced Engineering

Senior-level patterns for reliability and governance.

The Medallion Architecture

  • Bronze 🥉: Raw landing zone (Immutable).
  • Silver 🥈: Cleaned and integrated (The Truth).
  • Gold 🥇: Curated reporting layer (Business-ready).

Slowly Changing Dimensions (SCDs)

Managing historical accuracy. Type 2 is the standard for tracking history by adding new records with effective dates.
